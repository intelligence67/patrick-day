# Список змін для розробника

## Що було зроблено

- Збережено оригінальний дизайн лендингу `patricio.cuatrobet.com` без редизайну.
- Додано бізнес-логіку реєстрації у стилі стандартних CuatroBet-лендингів.
- Форма тепер підтримує:
  - вибір бонусу
  - телефон
  - email
  - пароль
  - чекбокс згоди
- Додано інтеграційний контракт для експорту в основний repo:
  - `#register-form`
  - `#selected-bonus-input`
  - `#selected-bonus-code`
  - `#register-form-error`
  - `.brand__logo`

## Що виправлено по UI

- Виправлено стрибки висоти картки форми при появі помилок.
- Помилки тепер показуються з анімацією, а не через різкий reflow.
- Повідомлення про помилки розташовані рівномірно між полями.
- Прибрано окремий текст помилки під паролем.
- Для пароля тепер використовується тільки існуючий хінт `Al menos 6 caracteres`, який змінює стан/колір при помилці.
- Вирівняно блок з чекбоксом, щоб він виглядав симетричніше.

## Що додано по інфраструктурі

- Додано `Vite`-оточення для локальної розробки.
- Додано `npm run build:repo` для експорту в основний repo.
- Додано `landing.export.json`, щоб цей лендинг експортувався в стандартний Bono-style shell.
- Оновлено спільний exporter у `../../template/scripts/build-repo.mjs`, щоб підтримувати режими експорту.

## Як працює інтеграція

- У сирому репозиторії лендинг може працювати локально і напряму викликати registration API.
- Після експорту в основний repo лендинг переходить на спільний CuatroBet flow через:
  - `/common/js/auth-helper-v2.js`
  - `/common/js/landing-welcome-adapter.js`
  - `marketing_lib_script.inc`
  - MTFEF callbacks

## Команди

```bash
npm install
npm run dev
npm run build
npm run build:repo
```

## Важливо

- Дизайн не змінювати без окремого запиту.
- Якщо потрібна подальша візуальна правка, робити її поверх поточного DOM і не ламати інтеграційні id/hidden-поля.
